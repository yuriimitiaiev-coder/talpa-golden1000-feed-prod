TALPA Golden1000 — виправлення статусу залишків v2

Проблема:
Prom.ua залишав 10 відсутніх товарів у статусі «В наявності», тому що
у fallback-картках не було тега quantity_in_stock. Під час імпорту Prom
міг зберігати попередній позитивний залишок і повертати товар у наявність.

Виправлення:
- available="false";
- quantity_in_stock завжди створюється зі значенням 0;
- видаляється можливий маркер in_stock;
- альтернативні поля кількості також обнуляються.

Перевірено локально:
- 1000 товарів у фінальному XML;
- 990 актуальних;
- 10 fallback-позицій;
- усі 10 мають available="false";
- усі 10 мають <quantity_in_stock>0</quantity_in_stock>.

Встановлення:
1. Розпакуйте ZIP.
2. У GitHub відкрийте репозиторій talpa-golden1000-feed-prod.
3. Відкрийте scripts.
4. Add file → Upload files.
5. Завантажте filter_feed.py із цього архіву та підтвердьте заміну.
6. Commit changes.
7. Actions → Build and publish Golden 1000 XML → Run workflow.
8. Після Success у Prom запустіть імпорт вручну один раз із полями
   «Наявність» і «Кількість» та увімкніть «Оновити примусово» лише для
   цього контрольного ручного запуску.
