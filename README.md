# TALPA Golden1000 Filter

Production-модуль для автоматичного оновлення каталогу TALPA на Prom.ua.

Кожні 4 години модуль:

1. завантажує актуальний повний XML Dropshipping.ua;
2. залишає лише 1000 затверджених артикулів із `golden_codes.txt`;
3. перевіряє, що всі 1000 артикулів знайдені й не дублюються;
4. залишає лише потрібні категорії та їхні батьківські категорії;
5. публікує `golden1000.xml` через GitHub Pages.

## Склад пакета

- `.github/workflows/publish.yml` — автоматичний запуск і публікація;
- `scripts/filter_feed.py` — фільтрація та контроль XML;
- `golden_codes.txt` — рівно 1000 затверджених артикулів;
- `docs/golden1000.xml` — початкова перевірена версія фіда;
- `docs/status.json` — технічний статус останньої збірки;
- `requirements.txt` — залежності Python;
- `SETUP_UA.md` — покрокове налаштування;
- `MANIFEST.txt` — контрольний перелік файлів.

## Безпека

Публікація не відбудеться, якщо:

- XML постачальника недоступний або пошкоджений;
- відсутній хоча б один Golden-артикул;
- знайдено дубль артикула;
- результат не містить рівно 1000 товарів.

У разі помилки попередня успішно опублікована версія GitHub Pages залишається доступною.

## Налаштування Prom

Після успішної публікації фіда:

- увімкнути оновлення **Код товару/Артикул**, **Ціна**, **Наявність**;
- не вмикати оновлення кількості — вихідний XML не містить окремого поля quantity;
- не оновлювати назви, описи, фото, групи, характеристики, знижки та особисті нотатки;
- `Тільки оновлення` — увімкнути;
- товари, яких немає у файлі — `Залишити без змін`;
- автоматичне оновлення — `Раз на 4 години`.

Детальна послідовність наведена в `SETUP_UA.md`.
